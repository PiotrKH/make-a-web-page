html basics

An HTML documents needs to follow a certain structure for it to work (to be correctly interpreted by web browsers):
  1. document type tag
  2. html tag / top level tag
  3. head tag:
    1. link tag (to .css style file)
    2. title tag
  4. body tag
The term side-way triangles refers to the branching structure of HTML pages, where:
- opening / closing tags can be either text or another element
- DOM - document object model, refer to the tree-like structure of a webpage

Front-end development means translating graphical form or design into HTML page - e.g. implementing/translating graphical design into a webpage.
HTML controls the structure of a website, while CSS controls its style (how the webpage looks).

boxes

Websites follow box design, simple, yet important concept. When building websites, we may think of giving them box-like structure, starting with/from big to small. Sketching this with pencil and paper usually helps with first stages of website building.
Boxes are defined with div tags, a block element, which can enclose multiple echelons of indented div-boxes in itself. By boxyfying website content, we can group the content in classes. This allows us to specify how particular classes should look like - this can be done with CSS. E.g. we can design how headers in each particular level look like, without having to do that every time in HTML code.

Hints for structuring a website:

  1. structure of the page, get it by boxyfying it, look for natural boxes
  2. size of the boxes, repeated styles of semantic
  3. where to position them on the page
  4. write the HTML
  5. apply styles (from BIGGEST to smallest)
  6. fix things/

css

Apart from putting the style in the style .css file, it is also possible to add a CSS style without linking to the CSS file, one can put the style between style tags.
With CSS the most specific rule is applied to every element - cascading, going down until the most specific of the rules applies inheritance in CSS from wikipedia:
Inheritance is a key feature in CSS, it relies on the ancestor-descendant relationship to operate. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants. Inheritance relies on the document tree, which is the hierarchy of (X)HTML elements in a page based on nesting. Descendant elements may inherit CSS property values from any ancestor element enclosing them. In general, descendant elements inherit text-related properties, but box-related properties are not inherited.

Properties that can be inherited are:

color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and - word-spacing.

Properties that cannot be inherited are:

background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index.

Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing the software developers to write less CSS. It enhances faster-loading of web pages by users and enables the clients to save money on bandwidth and development costs. selector - select them, either specifically e.g. p (paragraph) or my class (e.g. description)
